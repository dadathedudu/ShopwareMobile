Shopware Mobile ist ein mobiles Template für die eCommerce-Software Shopware. Shopware Mobile ist speziell für den mobilen Einkauf mit iOS, Android und BlackBerry Endgeräte konzipiert.

1. Informationen

Features der Beta-Version:

  • Mobile Version als Subshop nutzbar
  • Registrierung und Login von Kunden
  • Unterstützung von Bundles , Varianten-, Konfiguration- und Liveshopping-Artikeln
  • vollwertige Suche auf Basis der normalen Suche (keine Lizenz der Intelligenten Suchfunktion benötigt)
  • Abgabe von Kommentaren auf der Detailseite
  • Unterstützung von Bannern in Unterkategorien
  • Einkaufswelten Karussell auf der Startseite
  • Tracking von Bestellungen und Registrierungen über einen Subshop
  • Unterstützung für den Einsatz als nativen Applikation
  • Dynamische Berechnung des Warenkorbwert

Geplante Features für die Release-Version:

  • voll anpassbare Startseite
  • History-Support
  • Detailansicht im Warenkorb
  • eigenständiges Backend-Modul
  • Unterstützung der intelligenten Suche
  • Kunden-Center

Unterstützte Betriebssysteme:

  • iOS ab Version 3.1.2 (empfohlen iOS 4/iPhone 4)
  • Android ab Version 2.2 "Froyo"
  • BlackBerry OS ab Version 6

Enthaltende Farbtemplates:

Die Farbtemplates sind in der Plugin-Konfiguration von Shopware Mobile wählbar.

  • Android
  • Blue
  • Brown
  • Default
  • Green
  • Grey
  • iOS
  • Orange
  • Pink
  • Red
  • Turquoise

Bekannte Bugs:

  • Beim Einsatz als Subshop muss die Plugin-Konfiguration im jeweiligen Subshop vorgenommen werden
  • Nach dem Wechsel auf dem Warenkorb ist beim zweiten Laden die Detailseite vom Layout zerstört
  • PagingToolbar wird noch nicht aktualisiert, wenn die Anzahl der angezeigten Artikel verändert wurde
  • Nach Wechsel der Kategorieseite wird das Layout nicht aktualisiert, so dass der letzte Artikel im Listing nur abgeschnitten sichtbar ist
  • Es kann vorkommen, dass die Größe von Bannern nicht richtig bestimmt werden kann
  • Bei der Anmeldung im Checkout-Prozess kann im ersten Versuch die Bestellung nicht abgesendet werden, da die Session nicht richtig gesetzt ist
  • Filterung nach Formularen im "Informations"-Bereich wird noch nicht vorgenommen
  • Wenn kein Suchergebnis vorliegt sollte eine Meldung angezeigt werden
  • Bilderzoom bei Portaitbildern ist nicht sinnvoll
  • Text des Back-Buttons sollte ab 8-9 Zeichen gekürzt werden. jQuery Mobile hat hier eine reine CSS Lösung, welche adaptiert werden kann
  • Bei Orientierungswechsel auf der Startseite wird das Einkaufswelten Karussell nicht aktualisiert
  • Bei sehr langen Artikelnamen kann es vorkommen, dass der Preishinweis nur noch abgeschnitten sichtbar ist
  • Die Formatierung des Informationsbereichs ist noch nicht einheitlich mit dem Rest der Applikation
  • Die Auswahl des Herstellers sollte eine Gruppe von Radioboxen sein statt ein Selectfeld, da sonst der Eindruck erweckt wird, dass schon ein Hersteller gewählt ist
  • Im IE7 wird eine Fehlermeldung geworfen, wenn das Plugin aktiv ist

Bitte posten Sie alle weiteren Bugs, die nicht hier aufgelistet sind im Shopware Public Trac.

2. Installation und Einrichtung

Die Installation von Shopware Mobile gestaltet sich genauso einfach wie bei jeden anderen Plugin im Shopware Community Store.

Sie laden Sich das Plugin "Shopware Mobile" aus dem Store, laden es über Ihr Shopware Backend hoch und aktivieren es über den Plugin-Manager.

Plugin-Konfiguration:

Shopware Mobile als Subshop verwenden

Mit dieser Konfigurationsmöglichkeit legen Sie fest, ob Sie Shopware Mobile als Subshop nutzen.

Subshop ID

Mit dieser Konfigurationsmöglichkeit legen Sie die Subshop ID fest, die Shopware Mobile benutzen soll. Diese Einstellung ist abhängig von "Shopware Mobile als Subshop verwenden".

Unterstützte Geräte (mit Pipe getrennt)

Mit dieser Konfigurationsmöglichkeit legen Sie die unterstützten Endgeräte fest. Mögliche Endgeräte sind: Android, BlackBerry, iPhone, iPod und iPad. Bitte achten Sie darauf, dass die Endgeräte jeweils mit einen Pipe-Symbol getrennt werden müssen.

Shopseiten-Gruppen, die für den Informations-Bereich genutzt werden soll

Mit dieser Konfigurationsmöglichkeit legen Sie fest welche Shopseiten-Gruppe für den Informationsbereich von Shopware Mobile genutzt werden soll. Es wird empfohlen eine separate Gruppe anzulegen.

Sencha.io "Src" benutzen

Mit dieser Konfigurationsmöglichkeit legen Sie fest ob Ihre Bilder über den Cloud-Service "Src" ausliefern lassen möchten. "Src" bietet Ihnen die Möglichkeit Ihre Produktbilder automatisch an die Bildschirmgröße des Benutzers anzupassen. Weitere Informationen erhalten Sie auf der Seite von Sencha.

Gutscheineingabe auf der Bestellbestätigungsseite anzeigen

Mit dieser Konfigurationsmöglichkeit legen Sie fest ob die Gutscheineingabe auf der Bestellbestätigungsseite angezeigt werden soll. Bitte beachten Sie, dass durch diese Option die Länge der Bestellbestätigungsseite verlängert wird.

Newsletteranmeldung auf der Bestellbestätigungsseite anzeigen

Mit dieser Konfigurationsmöglichkeit legen Sie fest ob die Newsletteranmeldung auf der Bestellbestätigungsseite angezeigt werden soll. Bitte beachten Sie, dass durch diese Option die Länge der Bestellbestätigungsseite verlängert wird.

Kommentarfeld auf der Bestellbestätigungsseite anzeigen

Mit dieser Konfigurationsmöglichkeit legen Sie fest ob ein Kommentarfeld auf der Bestellbestätigungsseite angezeigt werden soll. Bitte beachten Sie, dass durch diese Option die Länge der Bestellbestätigungsseite verlängert wird.

Logo-Pfad

Mit dieser Konfigurationsmöglichkeit legen Sie den Pfad des Logo fest, welches auf der Startseite angezeigt werden soll. Bitte achten Sie darauf, dass das Logo nicht breiter als 320 Pixel ist.

Logo-Höhe

Mit dieser Konfigurationsmöglichkeit legen Sie die Höhe des Logo fest. Diese Einstellung wird nur wirksam, wenn die Einstellung "Logo-Pfad" gesetzt ist.

Icon - nur iOS

Mit dieser Konfigurationsmöglichkeit legen Sie den Pfad des Icon fest, welches angezeigt werden soll, wenn der Benutzer Ihren Shop zum Home-Screen hinzufügt. Diese Einstellung wird nur von iOS Endgeräten wirksam. Das Icon muss eine Größe von 72 Pixel x 72 Pixel aufweisen.

Glanz über Icon - nur iOS

Mit dieser Konfigurationsmöglichkeit legen Sie fest, ob ein Glanz-Effekt über Ihren Icon angezeigt werden soll. Diese Einstellung wird nur wirksam, wenn die Einstellung "Icon - nur iOS" gesetzt ist.

Startup Screen - nur iOS

Mit dieser Konfigurationsmöglichkeit legen Sie den Pfad des Startup Screens fest, welcher angezeigt wird, wenn der Benutzer den Shop über eine Verknüpfung auf dem Home-Screen startet. Das Bild muss in einer Größe von 320 Pixel x 460 Pixel vorliegen.

Link zur normalen Seite

Mit dieser Konfigurationsmöglichkeit legen Sie fest, ob ein Link zur normalen Version Ihres Shops auf der Startseite angezeigt werden soll.

Farbtemplate

Mit dieser Konfigurationsmöglichkeit legen Sie das Farbtemplate fest, welches für Shopware Mobile verwendet werden soll. Eine Übersicht aller verfügbaren Farbtemplates finden Sie hier.

Zusätzliche CSS-Eigenschaften

Mit dieser Konfigurationsmöglichkeit legen Sie weitere CSS-Eigenschaften fest, welche dann im HTML-Dokument ausgegeben werden. So haben Sie die Möglichkeit Shopware Mobile an Ihre persönlichen Vorlieben anzupassen.

3. Native Applikation

Shopware Mobile kann zukünftig auch als native App bereitgestellt werden. So können Sie als Shopbetreiber die App Stores von Apple, Android und co. als zusätzliches Marketinginstrument nutzen und sich dauerhaft auf den Smartphones Ihrer Kunden platzieren.

Um Shopware Mobile als native Applikation bereitstellen zu können, benötigen Sie mehrere Tools und ein fundiertes Verständnis von Web-Apps

Voraussetzungen:

  • ein Mac-System mit mindestens 10.6 Snow Leopard
  • eine Apple Developer Lizenz
  • Xcode 4 (http://itunes.apple.com/de/app/xcode/id422352214?mt=12)
  • PhoneGap 0.9.5 (http://phonegap.googlecode.com/files/phonegap-0.9.5.zip)

Grundlegende Schritte zur Erstellung einer nativen iOS Applikation:

  1. Xcode herunterladen und installieren
  2. PhoneGap herunterladen und das iOS Paket installieren
  3. Neues Projekt über das Terminal anlegen - Skript zur Erstellung von PhoneGap Projekten in Xcode 4
  4. Befehl: create_project.sh <Projektname> <Speicherort>
  5. "Shopware Mobile - Native App Paket" herunterladen und in das "www"-Verzeichnis des Projekts schieben. Der Download für dieses Paket steht in Kürze im Wiki bereit
  6. allgemeine Einstellungen zum Projekt bearbeiten und für den AppStore deployen

Projekteinstellungen vornehmen:

In den allgemeinen Projekteinstellungen von Xcode 4 können Sie die unterstützten Geräteorientierungen, das App-Icon für normale und Retina Displays und das Startup Bild hinterlegen. Des Weiterem finden Sie hier eine Eigenschaft namens "Identifier". In diesen Feld müssen Sie Ihren Hersteller-String hinterlegen, welches Sie von Apple gestellt bekommen. Der Herstellerstring von uns sieht beispielsweise so aus:

com.shopwareag.<AppName>

Darunter finden Sie die Einstellung "Deployment Target". Diese Einstellungen gibt die Firmware-Version an, welche mindestens für diese Applikation benötigt wird. Setzen Sie diese Einstellung auf "3.1.2".

Weitere Informationen zu Xcode 4 und deren Einstellungen finden Sie im Developer Center von Apple.
